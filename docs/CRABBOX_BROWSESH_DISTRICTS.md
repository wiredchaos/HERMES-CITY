# Crabbox + browse.sh District Map

## Public-safe placement

Crabbox and browse.sh are infrastructure districts in the Agentropolis/Hermes stack.

They do not replace the agent or the builder. They give the agent safer repeatable surfaces for navigation and execution.

## District roles

| District | Tool | Role |
|---|---|---|
| Browser District | `browse.sh` | Website muscle memory layer. |
| Execution District | `Crabbox` | Disposable remote machine layer. |
| Governance District | Logs, proofs, policies, evidence | Oversight and audit layer. |

## Standard wording

> browse.sh does not replace the agent.
>
> It gives agents reusable browser skills so web workflows can be repeated safely, consistently, and audibly.

> Crabbox does not replace the builder.
>
> It gives agents disposable execution environments so code can be tested, verified, and discarded without exposing the operator machine.
