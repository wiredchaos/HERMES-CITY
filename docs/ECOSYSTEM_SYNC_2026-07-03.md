## Boundary rules

HERMES-CITY may describe concepts, schemas, public coordination patterns, and non-sensitive operating principles.

HERMES-CITY must not publish:

- private keys or secrets
- private Agentropolis runtime code
- private wallet infrastructure
- client data
- undisclosed strategy
- proprietary prompts
- internal orchestration details
- capital-sensitive workflows
