# Hermes 20 Workflows Public Signal

This document captures the public signal from the "20 Hermes Agent workflows" pattern set and translates it into safe HERMES CITY language.

HERMES CITY is public. Agentropolis is private.

This document describes concepts, schemas, and operating principles only. It does not expose private implementation details, keys, production code, wallet infrastructure, client information, or undisclosed strategy.

## Public Signal

The core insight is simple:

```text
AI stops being a search box when it gets tools, memory, schedules, files, and receipts.
```

Hermes Agent patterns show a shift from one-off prompting toward operator workflows.

HERMES CITY tracks this because public agent-native commerce requires more than model access. It needs visible lanes for research, monitoring, publishing, release management, negotiation prep, memory recovery, and governed execution.

## Safety Boundary

Public workflows should be described by posture:

| Posture | Meaning |
| --- | --- |
| Read-only | Observe, collect, summarize, cite, and map evidence. |
| Draft-only | Produce material for human review before sending, publishing, or applying. |
| Alert-only | Monitor and notify when a meaningful condition is met. |
| Approval-required | Prepare or simulate action, then wait for explicit operator approval. |
| Elevated execution | Reserved for sensitive lanes such as releases, payments, wallet actions, credentials, or production changes. |

HERMES CITY treats elevated execution as a governance problem, not a convenience feature.

## Public Architecture Pattern

```text
Operator intent
  -> Hermes-style routing
  -> tool lane classification
  -> policy posture
  -> output verification
  -> audit receipt
```

The public takeaway is not "let agents do everything."

The public takeaway is:

```text
let agents do bounded work with clear tools, clear memory, clear authority, and clear receipts.
```

## Why This Matters

Most chatbot usage still lives at the shallow end:

- summarize this
- rewrite that
- brainstorm ideas
- answer a question

Operator workflows move into the deeper end:

- watch this source
- prepare this meeting
- compare these claims
- detect this change
- stage this article
- inspect this repo
- recover this decision
- generate this render
- prepare this release

That shift is the real public signal.

The model is only one layer.

The operating layer around the model is what makes agents useful.
