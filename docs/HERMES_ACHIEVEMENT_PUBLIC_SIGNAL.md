# Hermes Achievement Public Signal

## Purpose

HERMES CITY tracks Hermes achievements as a public coordination signal for the Agentropolis ecosystem.

This repository should expose safe, shareable architecture only. It should not publish private Agentropolis implementation details, private operator logs, wallet authority rules, or hidden achievement triggers.

## Public Position

Hermes achievements are useful because they show repeatable operator and agent behaviors:

- session continuity
- governed delegation
- tool discipline
- model-routing maturity
- credential safety
- audit receipt habits
- public infrastructure learning

## Boundary

```text
HERMES CITY = public signal
Agentropolis = canonical law
AGENTROPOLIS-CREATOR = foundry packages
AGENTROPOLIS-AGENT-MCP = validation and routing
```

HERMES CITY may summarize achievement concepts, badge tiers, public dashboard patterns, and operator education. It must not define secret requirements or grant execution authority.

## Public Badge Interpretation

```text
Copper
  onboarding and basic usage

Silver
  consistent visible behavior

Gold
  reliable build and documentation habits

Diamond
  advanced orchestration signal

Olympian
  city-scale operator maturity signal
```

## Public Rule

A badge is a signal, not a permission.

No public achievement display should imply that an agent may spend funds, access credentials, execute trades, change infrastructure, or bypass approval controls.

## Agentropolis Tie-In

The public layer should point builders toward the city pattern:

```text
achievement
  -> receipt
  -> auditability
  -> trust signal
  -> governed capability map
```

The private Root City OS decides which signals matter for production workflows.
